# Filmroad QA Checklist (Manual)

본 문서는 `tester2` 가 담당하는 자동 테스트 범위(백엔드 유닛/통합 + 프론트 vitest) 밖의
**사용자 경험(UX) 레벨 수동 검증** 을 위한 체크리스트다. 자동 테스트가 커버하는
API 응답 스키마, Store pure logic, 서비스 레이어 규칙은 여기서 재검증하지 않는다.

- 대상 빌드: `master` HEAD (자동 테스트 197+ 건 green 가정)
- 실행: `scripts/verify.sh` 통과 후 수동 QA 진입
- 마커: `- [ ]` 체크박스 = Pass 시 체크. 한 항목이라도 Fail 이면 이슈 티켓 생성
- 각 항목: **재현 단계 / 기대 결과** 한 줄 이내 요약

---

## 1. 환경 준비

- [ ] `./gradlew bootRun` 기동 후 `http://localhost:8080/actuator/health` → `{"status":"UP"}`
- [ ] 프론트 `npm --prefix frontend run dev` 기동 후 `http://localhost:5173` 로드, 콘솔 에러 0건
- [ ] `application.yml` 의 `kakao.maps.js-key` 설정 존재, 지도 페이지 콘솔에 SDK 로드 성공 로그
- [ ] `GOOGLE_CLIENT_ID/SECRET`, `KAKAO_CLIENT_ID/SECRET` 환경변수 주입 확인 (`echo $GOOGLE_CLIENT_ID` 비어있지 않음)
- [ ] SpotDataLoader 기동 로그 `Loaded N spots (target ~15,000)` 출력, `place` 테이블 카운트 >= 14,000
- [ ] DB 한글 필드(`regionLabel`, `name`) 깨짐 없음 (CP949 → UTF-8 변환 정상)
- [ ] `application-test.yml` 로 통합 테스트 별도 DB 분리 확인 (운영 데이터 오염 방지)
- [ ] JWT 쿠키 `ATOKEN` 이름/HttpOnly/SameSite=Lax 속성 DevTools 확인

**Pass 조건**: 위 8항목 모두 체크. **Fail 조건**: 기동 시 한글 깨짐 또는 place 카운트 < 10,000.

---

## 2. 스모크 플로우 (End-to-End 1회 관통)

신규 사용자가 앱 첫 실행 → 보상까지 한 번에 도달하는 해피패스. 5~8분 소요.

- [ ] 앱 첫 실행 시 온보딩(12) 진입, 미인증 상태에서 홈 진입 시 비로그인 CTA 노출
- [ ] Google 로그인 → 리다이렉트 복귀 → 홈(01) 정상 표시
- [ ] 홈에서 추천 카드 1개 탭 → 장소 상세(03) 진입, 지도 미니맵·작품 배지 모두 렌더
- [ ] 하단탭 지도(02) 이동 → 마커 군집 표시, 현재 위치 버튼 동작
- [ ] 지도에서 임의 마커 탭 → 바텀시트 → '상세 보기' → 03 이동
- [ ] 하단탭 카메라 CTA(04) → `getUserMedia` 권한 허용 → 프리뷰 노출
- [ ] 촬영 후 업로드(05) → 장소/작품 선택 → 성공 토스트
- [ ] 리워드(06) 이동 → 방금 업로드한 활동으로 포인트/배지 변동 확인
- [ ] 탐색(13) 피드 진입 → 업로드한 사진 최상단 노출 (정렬: 최신순)
- [ ] 프로필(07) → 스탬프북(08) 진입 → 신규 스탬프 해금 시각 효과
- [ ] 로그아웃 후 다시 홈 진입 시 미인증 상태로 복귀

**Pass 조건**: 11단계 끊김 없이 완주. **Fail 조건**: 어느 단계에서든 흰 화면/무한 스피너/500.

## 4. 상호작용 체크 (Like / Comment / Follow 트릴로지)

### Like

- [ ] 홈 카드 / 장소상세 / 피드 카드 / 갤러리 라이트박스 4곳의 좋아요 상태·카운트 동기
- [ ] 연타(5회) 시 최종 상태가 서버값과 일치 (토글 레이스 없음)
- [ ] 미인증 상태에서 좋아요 → 온보딩 리다이렉트, 재로그인 후 원상태 복귀

### Comment

- [ ] 피드 카드 댓글 아이콘 → CommentSheet 바텀시트 오픈
- [ ] 댓글 작성 시 리스트 맨 위 즉시 추가 + 카운트 +1
- [ ] 빈 댓글 제출 차단, 긴 댓글(500자+) 서버 검증 메시지 노출
- [ ] 시트 닫기 후 재오픈 시 최신 댓글 유지

### Follow

- [ ] 타 유저 프로필 팔로우 → 버튼 텍스트 '팔로잉' 전환
- [ ] 내 프로필에서 자기자신 팔로우 버튼 노출되지 않거나 401/403 방어
- [ ] 팔로우 후 FOLLOWING 피드 탭에 해당 유저 업로드 즉시 포함
- [ ] 언팔로우 시 FOLLOWING 피드에서 제거

**Pass 조건**: 4곳 동기, 레이스 없음, 자기자신 팔로우 차단 확인.

- [ ] 모든 아이콘 버튼에 `aria-label` 부여 (좋아요/저장/팔로우/댓글)
- [ ] 탭 이동(Tab 키)으로 모든 인터랙티브 요소 도달 가능
- [ ] 바텀시트 열림 시 포커스 트랩, ESC 키로 닫힘

- [ ] 이미지 `alt` 속성: 장식용은 `alt=""`, 의미 전달용은 기술

- [ ] `prefers-reduced-motion` 시 애니메이션 감소 또는 제거

## 참고: 누적 디버트 영역 (알려진 이슈)

QA 는 아래 항목을 '알려진 이슈'로 인지하고, 별도 신규 티켓 대신 기존 이슈에 코멘트 누적:

- **FrBrandHeader 중복 렌더**: 일부 페이지 네비게이션 경로에서 헤더가 2번 렌더되는 레거시 흔적 (해소 중). 신규 페이지 추가 시 상위 레이아웃에서 헤더가 이미 렌더되는지 확인.
- **CurrentUser DEMO 폴백**: 미인증 상태에서 일부 컴포넌트가 `DEMO_USER` 목 객체를 사용. 실계정 전환 시 상태 오염 여부 주기적 확인.
- **regionLabel 파생 불일치 (해소 중)**: 백엔드 파생 vs 프론트 파생 경로 2종 혼재. 한 화면 안에서만 파생 경로 사용하는지 검증.
- **SpotDataLoader CP949**: 인코딩 변환 실패 시 일부 한글 레이블이 `?` 로 표기되는 케이스 보고됨. place 테이블 샘플 100건 수동 검수 권장.
- **OAuth 콜백 경합**: 느린 네트워크에서 콜백 처리 중 사용자가 뒤로가기 누를 시 세션 반쪽 생성 의심. 재현 시 네트워크 탭 HAR 첨부.
- **지도 줌아웃 시 군집 갱신 지연**: 마커 수 많을 때 군집 재계산 350~500ms. 체감상 수용 가능하나 추후 web worker 이전 검토.
